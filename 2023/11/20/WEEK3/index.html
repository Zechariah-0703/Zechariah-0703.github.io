WEEK 3 周报

这周搞小组作业，也没什么进展，就弄了个博客。

GEEK结算周，接着坐牢。下周的周报会写一些GEEK的Write up.

张灏 2023/11/20

从零开始的博客生活

此博客已经具备的功能

  • 搜索

  • 打赏

  • 抽象语录循环播放

  • 图片懒加载

  • 版权提醒

  • 如果404了，那前面的区域以后再来探索吧

一.环境准备

node环境

  • 官网安装 nodejs，因为Hexo是基于 Node.js 驱动的一款博客框架，路径改D盘，全程next。

git 环境

  • 官网安装git，路径改D盘，全程next。

配置环境变量

  • node安装时环境变量自动配好了。（爽到）

  • git的环境变量出的问题最多，配置了两次，最后用户变量里留了2条，系统变量里留了3条，一共五条。（不敢删，能用就是成功）

检测

在cmd里使用如下命令查看两者版本

node -v 
git -v

输出版本号就是成功。

二.创建Github仓库

选择public

仓库取名：用户名.github.io

三.本地配置安装Hexo

  1. D盘中创建一个用来存放Github本地仓库文件的目录Blog

  2. 安装：输入命令：cnpm install -g hexo-cli

  3. 初始化Hexo：输入命令：hexo init

四.设置ssh

  1. 生成sshkey

  2. 首先在git目录下输入命令：

    ssh-keygen -t rsa -C ‘注册时的邮箱地址’

    接着按四下回车

  3. 接着去查看C盘目录(此文件夹所在C:\用户\用户名目录下)，然后在记事本中打开id_rsa.put文件并复制生成的key

  4. 将key配置到GitHub中

  5. 输入命令：

    git config --global user.name “注册时用户名”
git config --global user.email “注册时邮箱”

五.上传博客

  1. 首先找到之前创建仓库生成的Github仓库Http或SSH地址
    打开blog文件夹下的_config.yml
    修改以下内容：
deploy:
​    type: git
​    repo: git@github.com:用户名/用户名.github.io.git
​    branch: main

  1. 安装上传工具

    输入命令：

    cnpm install hexo-deployer-git

  2. 新建文章

    输入命令：hexo new “文章名称”

    然后会发现source文件夹里会出现一个文章名称.md

  3. 经典三件套

    输入命令：

hexo g（生成）

hexo s（本地预览）

hexo d（部署）

六.访问博客

浏览器输入访问用户名.github.io 就可以访问博客啦

剩下的就是美化过程

如果你看到这篇文章的话，那我应该美化的差不多了。
